What is Ensemble Learning?

Core Concept: Combine multiple models to achieve better performance than any individual model

  • Why it works: Reduces both bias & variance through model diversity
  • Wisdom of crowds: Multiple "opinions" lead to better decisions
  • Error reduction: Individual model errors tend to cancel out

Majority Voting

Definition

A simple ensemble method where multiple models vote on the final prediction. For classification, the class with the most votes wins. For regression, predictions are averaged.

Examples

Simple Classification

3 classifiers predict: [Cat, Dog, Cat] → Final: Cat

Medical Diagnosis

Multiple expert opinions combined

Spam Detection

Multiple algorithms vote on email classification

Pseudocode

# Majority Voting Pseudocode Algorithm: Majority Voting Ensemble Input: Training set D, Test instance x, Base models M1, M2, ..., Mn 1. FOR each base model Mi: a. Train Mi on dataset D b. Store trained model 2. FOR test instance x: a. predictions = [] b. FOR each trained model Mi: i. pred_i = Mi.predict(x) ii. predictions.append(pred_i) c. IF classification: final_prediction = mode(predictions) # Most frequent class d. ELSE IF regression: final_prediction = mean(predictions) # Average value 3. RETURN final_prediction

Bagging (Bootstrap Aggregation)

Definition & Process

Bootstrap Aggregation trains multiple models on different bootstrap samples (sampling with replacement) and combines their predictions.

Process Steps

  • Create bootstrap samples from original dataset
  • Train one model on each bootstrap sample
  • Combine predictions through voting/averaging

Advantages

  • Reduces overfitting
  • Decreases variance
  • Can be parallelized
  • Works well with high-variance models

Examples

  • Random Forest (bagging + feature randomness)
  • Bagged Decision Trees
# Bagging (Bootstrap Aggregation) Pseudocode Algorithm: Bootstrap Aggregation Input: Training set D, Number of models n, Base algorithm A 1. trained_models = [] 2. FOR i = 1 to n: a. D_i = bootstrap_sample(D) # Sample with replacement b. M_i = train_model(A, D_i) c. trained_models.append(M_i) 3. FOR test instance x: a. predictions = [] b. FOR each model M_i in trained_models: i. pred_i = M_i.predict(x) ii. predictions.append(pred_i) c. IF classification: final_prediction = majority_vote(predictions) d. ELSE IF regression: final_prediction = average(predictions) 4. RETURN final_prediction

Boosting

Definition

Sequential ensemble method where each model learns from the mistakes of previous models by adjusting sample weights.

Process Steps

  • Train first model on original data
  • Increase weights of misclassified samples
  • Train next model focusing on hard examples
  • Repeat until desired number of models
  • Combine models with weighted voting

Examples

  • AdaBoost (Adaptive Boosting)
  • Gradient Boosting
  • XGBoost, LightGBM
# Boosting (AdaBoost) Pseudocode Algorithm: Adaptive Boosting Input: Training set D = {(x1,y1),...,(xm,ym)}, Number of rounds T 1. Initialize sample weights: w_i = 1/m for all i 2. FOR t = 1 to T: a. Train weak learner h_t using weights w b. Calculate error: ε_t = Σ(w_i * I(h_t(x_i) ≠ y_i)) c. IF ε_t > 0.5: BREAK d. Calculate model weight: α_t = 0.5 * ln((1-ε_t)/ε_t) e. Update sample weights: FOR i = 1 to m: IF h_t(x_i) = y_i: w_i = w_i * exp(-α_t) # Decrease weight for correct ELSE: w_i = w_i * exp(α_t) # Increase weight for incorrect f. Normalize weights: w_i = w_i / Σ(w_j) 3. FOR test instance x: final_prediction = sign(Σ(α_t * h_t(x))) 4. RETURN final_prediction

Stacking

Definition

Meta-learning approach that uses a meta-model to learn how to best combine predictions from multiple base models.

Architecture

  • Level-0: Base models trained on original data
  • Level-1: Meta-model trained on base model predictions
  • Uses cross-validation to avoid overfitting

Examples

  • Combining different algorithms (SVM, Random Forest, Neural Networks)
  • Kaggle competitions often use stacking
# Stacking Pseudocode Algorithm: Stacked Generalization Input: Training set D, Base models M1,...,Mn, Meta-model Meta 1. TRAINING PHASE: a. Split D into k folds for cross-validation b. level_0_predictions = [] c. FOR each base model M_i: i. cv_predictions = [] ii. FOR each fold j: - Train M_i on (k-1) folds - Predict on fold j - cv_predictions.append(predictions) iii. level_0_predictions.append(cv_predictions) d. Create meta-dataset from level_0_predictions e. Train Meta model on meta-dataset 2. PREDICTION PHASE: FOR test instance x: a. base_predictions = [] b. FOR each trained base model M_i: i. pred_i = M_i.predict(x) ii. base_predictions.append(pred_i) c. final_prediction = Meta.predict(base_predictions) 3. RETURN final_prediction

Key Takeaways & When to Use Each Method

Key Takeaways

  • Ensemble methods combine multiple models to achieve better performance than individual models
  • Majority Voting is simplest - just combine equal-weight predictions
  • Bagging reduces variance through bootstrap sampling and parallel training
  • Boosting reduces bias through sequential learning from errors
  • Stacking uses meta-learning to optimally combine diverse models
  • Choose method based on your models' characteristics and computational constraints

When to Use Each Method

Majority Voting

When you have multiple well-performing models of similar quality

Bagging

When your base models have high variance (like deep decision trees)

Boosting

When your base models have high bias (weak learners like shallow trees)

Stacking

When you have diverse, heterogeneous models and computational resources
